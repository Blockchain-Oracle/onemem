# ERC-8004 — Trustless Agents

## What it is

ERC-8004 ("Trustless Agents") is an **Ethereum Improvement Proposal** that defines a standard for **AI agent identity, reputation, and validation** on EVM chains. It went live on Ethereum mainnet on January 29, 2026. It is the EVM-side equivalent of "Sui has Move objects for agents." Importantly, it positions itself as a **trustless extension of Google's A2A (Agent-to-Agent) Protocol** for on-chain agents.

## The three registries

ERC-8004 defines three lightweight on-chain registries:

### 1. Identity Registry
- Each agent is an ERC-721 NFT.
- The NFT points to an **agent card** (structured JSON): name, capabilities, service endpoints (MCP, A2A, web), payment address.

### 2. Reputation Registry
- Standardized format for reputation feedback after agent interactions.
- Reputation data is portable across applications.
- Per-agent, per-skill, per-task.

### 3. Validation Registry
- On-chain validation of agent outputs.
- Anchors proofs from off-chain validators (TEE attestations, ZK proofs, multi-party verification).

## Cross-chain support

- For EVM chains: namespace `eip155`, `chainId` = the chain's ID (1 for Ethereum, 43114 for Avalanche C-Chain, etc.).
- Native to any EVM chain.
- Adoption on Base, Optimism, Arbitrum where gas is low enough for frequent micropayments and reputation submissions.

## Verifiability mechanism

- **The standard itself doesn't prescribe.** ERC-8004 anchors *pointers to* validation proofs — the proof mechanism is pluggable: TEE attestations, ZK, MPC, cryptoeconomic.
- This is a **registry + interop standard**, not an execution / verification engine.

## Status

- Live on Ethereum mainnet (Jan 29, 2026).
- Registry Broker is one of the first launch tools.
- Multi-chain adoption underway on Base, Optimism, Arbitrum, Avalanche.

## How OneMem complements ERC-8004

- **Different chain, different scope, deeply complementary.** ERC-8004 is EVM, registry-focused, agent-identity-focused. OneMem is Sui, content-focused, agent-memory-focused.
- **ERC-8004 agent cards reference service endpoints (MCP, A2A, web).** An ERC-8004 agent card could reference an **OneMem MemoryNamespace** as one of its endpoints — making cross-chain agents discoverable on EVM but reading/writing memory on Sui via OneMem.
- **Reputation cross-reference.** An ERC-8004 agent's reputation score could be computed in part from its OneMem trace history — verifiable proof of past behavior anchored on Sui, reputation aggregated on Ethereum.
- **The "trustless extension of A2A" framing matters.** Google's A2A is the agent-communication standard. ERC-8004 trustlessifies the identity layer. OneMem could be the third leg: trustless memory + trace layer. Together: A2A (comms) + ERC-8004 (identity/reputation) + OneMem (memory/trace) = a full Web3 agent stack.
- **Specifically for Sui:** ERC-8004 doesn't currently support non-EVM chains (the `eip155` namespace assumption). A **Move-equivalent of ERC-8004** could emerge from the Sui ecosystem — or Sui agents could be represented in ERC-8004 via a bridge contract that maintains an EVM mirror of a Sui agent's identity object.
