Where everything is.

A complete map of the OneMem campaign assets in this project — what each file is, how to open it, how to edit it, and what's left to do (the audio). Built to be downloaded with the project and handed to a local agent.

One-line summary. Three deliverables: a Brand Kit (5 static graphics on a canvas), a 30-second Launch Video (silent — needs scoring), and a Sound Design beat-map spec. Everything is plain HTML/JS that runs in a browser; no build step.

04 — AUDIOWhat's left for your agent

The video ships silent on purpose. All sound code was removed so you have a clean slate to score it locally. There is no audio playing in Launch Video.html right now.

Everything the audio agent needs:

  • The brief / beat map: Sound Design.html (or video/OneMem Launch Video - Audio Spec.md) — cue timings, loudness targets (−14→−10 dB mean), and the sonic palette, beat by beat.
  • Reference tracks (the “bar” for energy/feel) are in uploads/: triton_one_…720.mp4, SuiNetwork_…720.mp4, agentcardai_…1080.mp4. Note agentcardai has a spoken voiceover baked in; triton/Sui are music-led.
  • Optional starting point: video/audio.jsx still contains a Web-Audio score engine (rhythmic synth + a real-file bed) — it is not loaded by the video. Re-add <script src="video/audio.jsx"> and an <AudioBed src=…> tag if you want it back, but the intent is for you to lay a proper licensed track.

Copyright note: the three reference MP4s are other companies' launch videos. Use them to match tempo/energy only — don't ship their audio in the public OneMem cut. Bring a licensed or original track.

05 — RUNNING LOCALLYHow to open these

  • These are static HTML files with no build step — but they load React, Babel, and Google Fonts from a CDN, so keep an internet connection the first time.
  • They reference assets by relative path (e.g. kit/parts.jsx, packages/brand/vendor-logos/svg/…, _ds/…), so serve from the project root, don't move the HTML files out on their own. A one-liner like npx serve or python3 -m http.server in the project root works; then open the file in a browser.
  • Opening via file:// may block the .jsx fetches in some browsers — prefer the local server.
