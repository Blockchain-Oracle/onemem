# Runtimes

Native plugins and MCP integrations for coding-agent runtimes.

## Claude Code

```bash
claude plugin marketplace add Blockchain-Oracle/onemem
claude plugin install onemem@onemem
```

Hooks `SessionStart`, `PostToolUse`, and `SessionEnd` record a Merkle-chained
TraceSession per session when the plugin is installed, configured, and hooks run.
The GitHub repository marketplace path is current, and
`@onemem/claude-code-plugin@0.1.0` is current on npm. Trusted live Claude Code
hook proof remains a separate on-chain verification boundary.

## Codex

OneMem ships a Codex plugin package at `packages/plugin-codex`. It bundles the
OneMem MCP server for memory, search, replay, share, and verification tools.

```bash
codex plugin marketplace add Blockchain-Oracle/onemem --json
codex plugin add onemem-codex@onemem --json
```

Optional Codex hook scripts are included and flush through the published
`@onemem/sdk-ts@0.6.0` trace CLI, but full automatic Codex tool-call trace
coverage is only claimed after a trusted `/hooks` session emits a verifiable
on-chain OneMem `TraceSession`. Local Codex CLI 0.140 `codex exec` testing did
not run hooks, so it is not treated as that proof path. MCP tools work without
hook trust.

## OpenClaw

```bash
openclaw plugins install @onemem/oc-onemem
npx @onemem/oc-onemem init
openclaw gateway run
```

Loads `@onemem/oc-onemem` in `registrationMode: "full"`; records tool calls + agent output via the gateway.

## Hermes (Python)

```bash
pip install hermes-onemem
```

Subclasses Hermes's `MemoryProvider` ABC; buffers turns + tool calls + delegations and flushes one verifiable trace per run via the `onemem-trace` bridge.

<Note>
Registry note, 2026-06-18: runtime npm/PyPI packages in this repo are current
after `pnpm registry:status --strict`. MCP works without hook trust; automatic
Claude/Codex hook tracing still requires trusted live client sessions and
on-chain TraceSession verification before claiming full coverage.
</Note>

<Note>
All runtime integrations are defensive: a OneMem failure should never break your
agent. Trace-capable integrations buffer locally and avoid silent loss when an
on-chain write cannot be confirmed.
</Note>
