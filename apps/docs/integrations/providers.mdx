# Framework providers

Wrap your framework's model/agent/session and OneMem records each run as a verifiable on-chain trace. Provider wrappers are defensive by default: they buffer, flush off the host's critical path, and keep package-specific publish and live-test proof separate from the integration API.

<Note>
Registry note, 2026-06-18: explicit memory helper APIs are current on public
registry artifacts for all five providers:
`@onemem/vercel-ai-provider@0.1.2`, `@onemem/openai-agents@0.1.3`, and Python
providers `0.1.1`. This was verified with `pnpm registry:status --strict`;
re-run it before making a fresh npm/PyPI install claim.
</Note>

## JavaScript / TypeScript

<CodeGroup>
```ts Vercel AI SDK
import { withOneMem } from "@onemem/vercel-ai-provider";
const model = withOneMem(openai("gpt-4o"), { agentId: "my-agent" });
```

```ts OpenAI Agents
import { attachOneMem } from "@onemem/openai-agents";
attachOneMem(runner, { agentId: "my-agent" });
```
</CodeGroup>

## Python

<CodeGroup>
```python CrewAI
from crewai import Crew
from onemem_crewai import OneMemTracer, create_onemem_memory

tracer = OneMemTracer(agent_id="crew")
memory = create_onemem_memory()
prompt = memory.recall_context("Plan the task")
crew = Crew(
    agents=[...],
    tasks=[...],
    step_callback=tracer.step,
    task_callback=tracer.task,
)
crew.kickoff()
memory.capture("Crew result: ...")
tracer.flush()
```

```python LiveKit
from onemem_livekit import OneMemTracer, create_onemem_memory
memory = create_onemem_memory()
context = memory.recall_context("What does this caller prefer?")
OneMemTracer(agent_id="voice").attach(session)
```

```python ElevenLabs
from onemem_elevenlabs import OneMemTracer, create_onemem_memory
tracer = OneMemTracer(agent_id="voice")
memory = create_onemem_memory()
context = memory.recall_context("What should the voice agent remember?")
conv = Conversation(client, agent_id, requires_auth=True,
                    client_tools=tracer.wrap_tools(ClientTools()),
                    **tracer.callbacks())
```
</CodeGroup>

The Python providers shell out to the `onemem-trace` Node CLI for the on-chain trace write (Walrus/Seal are JS-first). Their explicit memory helpers use `onemem-sdk-python`'s `MemoryClient` bridge for MemWal recall/capture and stay defensive when memory is disabled or unavailable.

<Note>
Every provider is fire-and-forget and never breaks the host run. JS providers guard their listeners; Python providers flush off-thread and retain on unconfirmed writes.
</Note>
