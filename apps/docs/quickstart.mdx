# Quickstart

## 1. Install the CLI

```bash
npm install -g @onemem/cli
```

<Note>
Publication note, 2026-06-19: the published `@onemem/cli@0.6.3` install path is
verified. `npm exec --yes --package @onemem/cli@0.6.3 -- onemem --version`
prints `0.6.3`, and `pnpm release:preflight --strict --timeout 30` plus
`pnpm registry:status --strict` both pass.
</Note>

## 2. Provision (zero-config)

`init` resolves a signer (env key → Sui keystore → a generated, persisted wallet) and provisions a namespace + ReadWrite cap on testnet — no browser, no manual setup.

```bash
onemem init
```

It prints the namespace + cap and the env vars any runtime can use:

```text
✓ OneMem ready
  namespace   0x18451c…
  rwCap       0xcaa865…
```

## 3. Instrument your runtime

Pick your stack — each is one line. Trace-capable integrations record after the
runtime plugin/provider is installed, configured, and trusted by that runtime.
MCP tools are available without hook trust.

<Note>
Provider examples below are available from the current npm/PyPI artifacts after
`pnpm registry:status --strict`. Re-run that command before making a fresh
published-helper claim.
</Note>

<CodeGroup>
```bash Claude Code
claude plugin marketplace add Blockchain-Oracle/onemem
claude plugin install onemem@onemem
```

```bash Hermes (Python)
pip install hermes-onemem
```

```ts Vercel AI SDK
import { withOneMem } from "@onemem/vercel-ai-provider";
const model = withOneMem(openai("gpt-4o"));
```

```ts OpenAI Agents
import { attachOneMem } from "@onemem/openai-agents";
attachOneMem(runner);
```
</CodeGroup>

## 4. Verify

After your agent runs, verify any session - from the terminal or the public web page.

```bash
onemem verify <session-id>
```

```text
✓ VERIFIED
  agent      my-agent (production)
  status     Completed
  calls      12
  merkleRoot 0x2ec8c2…
```

Or share `app.onemem.xyz/verify/<session-id>` — anyone can confirm the chain, no login.
