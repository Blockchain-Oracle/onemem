# Quickstart

## 1. Install the CLI

```bash
npm install -g @onemem/cli
```

<Note>
Registry note, 2026-06-18: this command is the target public install path.
`pnpm registry:status` currently reports `@onemem/cli` and
`@onemem/dashboard` as missing from npm, so use a built checkout until registry
publication catches up.
</Note>

## 2. Provision (zero-config)

`init` resolves a signer (env key → Sui keystore → a generated, persisted wallet) and provisions a namespace + ReadWrite cap on testnet — no browser, no manual setup.

```bash
onemem init
```

It prints the namespace + cap and the env vars any runtime can use:

```text
✓ OneMem ready
  namespace   0x18451c…
  rwCap       0xcaa865…
```

## 3. Instrument your runtime

Pick your stack — each is one line. Traces start recording automatically.

<Note>
Provider examples below describe current source. Some helper APIs are in local
patch versions that public npm/PyPI artifacts have not caught up to yet; check
`pnpm registry:status` before treating them as published install proof.
</Note>

<CodeGroup>
```bash Claude Code
claude plugin marketplace add Blockchain-Oracle/onemem
claude plugin install onemem@onemem
```

```bash Hermes (Python)
pip install hermes-onemem
```

```ts Vercel AI SDK
import { withOneMem } from "@onemem/vercel-ai-provider";
const model = withOneMem(openai("gpt-4o"));
```

```ts OpenAI Agents
import { attachOneMem } from "@onemem/openai-agents";
attachOneMem(runner);
```
</CodeGroup>

## 4. Verify

After your agent runs, verify any session — from the terminal or the public web page.

```bash
onemem verify <session-id>
```

```text
✓ VERIFIED
  agent      my-agent (production)
  status     Completed
  calls      12
  merkleRoot 0x2ec8c2…
```

Or share `app.onemem.ai/verify/<session-id>` — anyone can confirm the chain, no login.
